# AI-Driven E-Commerce Demand Forecasting Dashboard
### AI活用によるECサイトの「物流・経営・在庫」最適化支援ダッシュボード

## プロジェクト概要
実際のECサイトのトランザクションデータを用いて、機械学習および統計的手法による需要予測モデルを構築。
**「経営層」「物流担当」「在庫担当」** という異なるステークホルダーが必要とする数値を可視化する、インタラクティブなダッシュボードアプリです。

** [デモアプリを開く](https://e-commerce-data-forecast-hhcflxc737vqrreum5y8em.streamlit.app/%EF%BC%89)** ## データセット
* **出典:** Kaggle - [E-Commerce Data](https://www.kaggle.com/datasets/carrie1/ecommerce-data)
* **内容:** イギリスを拠点とするオンラインギフトショップの全取引データ（主な顧客はグローバルな卸売業者）
* **期間:** 2010/12/01 〜 2011/12/09 (約1年間)
* **データ規模:** 約54万レコード
* **使用カラム:** InvoiceNo, StockCode, Quantity, InvoiceDate, UnitPrice, Country, Description, CustomerID

##  解決するビジネス課題 (3つのシナリオ)

本アプリケーションは、各担当者が必要とする「事実（数値とトレンド）」を直感的に提供します。

### 1. 物流担当者向け：注文数と変動幅の可視化
* **課題:** 繁忙期（11月〜12月）の注文急増に対し、客観的な予測値がないとシフト計画が立てられない。
* **実装機能:** 「注文数（Order Count）」を予測。予測線に加え、**予測の変動幅（バンド）** を表示することで、ピーク時にどれくらい上振れする可能性があるかを可視化します。

### 2. 経営層向け：予算対比のモニタリング
* **課題:** 売上目標に対して、現在のペースが順調なのか未達なのかが即座に判断できない。
* **実装機能:** 全社の「売上金額（Sales）」を予測。グラフ上に**「予算目標ライン（赤線）」** を明確に表示し、AI予測値との位置関係を一目で把握できるようにしました。

### 3. 在庫担当者向け：SKU別需要の予測
* **課題:** 数千種類ある商品（SKU）のうち、特定商品の需要トレンドが掴めない。
* **実装機能:** 任意のSKUを選択し、「販売個数（Quantity）」を予測。物流同様に変動幅（バンド）を表示し、需要のボラティリティ（安定度）を確認できます。

## アプリケーションの機能

###  2つの予測モード
用途に合わせて、サイドバーでモードを切り替え可能です。
1.  **精度検証モード (Backtest Validation):**
    * 過去のデータを学習・テスト用に分割し、AIの予測精度（MAPE/Accuracy）を検証します。実測値と予測値を重ねて表示します。
2.  **未来予測モード (Future Prediction):**
    * 全期間のデータを学習し、未知の未来（向こう30日間）を予測します。

### 3つの予測モデル
データの特性に合わせて、回帰モデルと時系列モデルを選択可能です。

| モデル | 分類 | 特徴・使い分けの想定 |
| :--- | :--- | :--- |
| **Random Forest** | 回帰 (機械学習) | **複雑なパターンの学習に強み。**<br>曜日やラグ（過去の売上）などの特徴量を組み合わせて学習します。非線形な関係性を捉えるのが得意です。 |
| **Prophet** | 時系列 (Facebook製) | **トレンドと季節性の分解に強み。**<br>「年周期」「週周期」などを考慮し、人間が解釈しやすい予測を行います。欠損値やノイズに頑健です。 |
| **ARIMA** | 時系列 (統計モデル) | **直近のトレンド維持に強み。**<br>自己相関（過去との相関）をベースに数理的に予測します。短期的な予測において安定した性能を発揮します。 |

# AI-Driven E-Commerce Demand Forecasting Dashboard
### AI活用によるECサイトの「物流・経営・在庫」最適化支援ダッシュボード

## プロジェクト概要
実際のECサイトのトランザクションデータを用いて、機械学習による需要予測モデルを構築。
単なる数値予測にとどまらず、**「経営層」「物流担当」「在庫担当」** という異なるステークホルダーの意思決定を支援するためのインタラクティブなダッシュボードアプリを開発しました。

**[デモアプリを開く]（https://e-commerce-data-forecast-hhcflxc737vqrreum5y8em.streamlit.app/）** 

## 📊 データセット
* **出典:** Kaggle - [E-Commerce Data](https://www.kaggle.com/datasets/carrie1/ecommerce-data)
* **内容:** イギリスを拠点とするオンラインギフトショップの全取引データ（対象顧客は、グローバルな卸売業）
* **期間:** 2010/12/01 〜 2011/12/09 (約1年間)
* **データ規模:** 約54万レコード
* **使用カラム:** InvoiceNo, StockCode, Quantity, InvoiceDate, UnitPrice, Country ,Description, CustomerID

## 解決するビジネス課題 (3つのシナリオ)

本アプリケーションは、以下の3つの業務課題を解決するために設計されています。

### 1. 物流担当者向け：クリスマス繁忙期のシフト最適化
* **課題:** 11月〜12月の繁忙期は注文が急増する。予測を見誤ると、人員不足による配送遅延（クレーム）や、過剰配置による人件費の無駄が発生する。
* **解決策:** 「注文数」を高精度に予測し、必要な作業量を算出。
* **実装機能:** 単なる予測グラフだけでなく、処理能力に基づいた**「推奨スタッフ人数」**を自動算出して提示。

### 2. 経営層向け：年末着地見込みの早期把握
* **課題:** 売上目標の達成可否が不透明で、追加の広告投資や予算修正の意思決定が遅れる。
* **解決策:** 全社の「売上金額」を予測し、現在のペースを可視化。
* **実装機能:** グラフ上に**「予算目標ライン」**を表示し、AI予測値との乖離（上振れ/下振れ）を強調表示して迅速な判断を促す。

### 3. 在庫担当者向け：過剰在庫と機会損失の最小化
* **課題:** 季節性商品は、売り切れ（機会損失）と売れ残り（不良在庫）のリスク管理が難しい。
* **解決策:** 特定の商品（SKU）ごとの「販売個数」を予測。
* **実装機能:** 予測線に対し、**「在庫切れリスク帯」と「過剰在庫リスク帯」**（バンド）を表示し、発注・セールの判断材料を提供。
